# adaptive-loudness
адаптивная громкость на компьютере, для Линукс, C++
Панель «Параметры»

🎯 Целевой уровень (dB) = -15,72

Это «эталон» — какую громкость должен слышать микрофон.


Каждую секунду программа измеряет, что слышит микрофон в dB
Если слышит тише этого числа → поднимает громкость sink
Если громче → опускает
Значение задаётся вручную или автоматически при калибровке
Шкала: 0 dB = максимум, -90 dB = тишина. Чем число выше (ближе к 0), тем громче эталон

🔉 Громкость min (%) = 20

Нижний предел. Программа никогда не опустит громкость sink ниже 20%, даже если контент очень громкий. Защита от того, что станет «вообще не слышно».


🔊 Громкость max (%) = 56

Верхний предел. Программа никогда не поднимет выше 56%. Защита ушей/колонок и от обратной связи микрофон↔колонки (если задерёт слишком высоко — пойдёт самовозбуждение).


⏱ Длительность замера (с) = 0,40

Сколько секунд аудио берётся для одного измерения RMS.


Меньше (0,1) — быстрая реакция, но «дёрганые» значения
Больше (0,5) — стабильные, но медленная реакция
0,4 — компромисс. Усреднение по 400 мс отфильтровывает мгновенные пики и провалы

🔄 Интервал обновления (с) = 0,50

Пауза между циклами регулировки. Раз в полсекунды программа: замеряет → считает → меняет громкость. Меньше число = быстрее реагирует, но больше нагрузка на CPU.


🔇 Порог тишины (dB) = -55

Граница, ниже которой считается «звук не играет».


Если из колонок (.monitor) слышится тише -55 dB → программа ничего не делает (показывает «Тишина»)
Это критично! Иначе в паузе диалога программа разогнала бы громкость на максимум
Если значение слишком высокое (-40) — программа пропустит тихие сцены
Если слишком низкое (-70) — может считать электрический шум за «звук играет»

📐 Макс шаг за тик (%) = 2,00

Максимальное изменение громкости за одну итерацию (0,5 с).


2% за полсекунды = 4%/сек — плавно, незаметно для уха
Если поставить 100% — будут резкие скачки, но мгновенная реакция
Это «тормоз», предотвращающий резкие прыжки громкости

〰 Сглаживание (0…1) = 0,35

Коэффициент инерции в формуле «новая громкость».

0.0 — никаких изменений (отключено)
0.35 — берётся 35% от расчётного «идеального» изменения, остальные 65% — текущее значение. Плавно
1.0 — берётся 100% расчётного значения (без сглаживания, максимальная резкость)

Формула: новая = текущая + (расчётная − текущая) × сглаживание


👆 Дельта ручной подстройки (%) = 5,00

Порог, при котором программа понимает «пользователь крутанул громкость сам».


Если громкость sink изменилась более чем на 5% и не от программы — программа считает, что вы передвинули регулятор в трее, и запоминает новое значение как новую цель
Меньше число = чувствительнее к ручным изменениям
Используется в адаптивном режиме (не в режиме «Фиксировать громкость»)


Панель «Статус»

🎤 Микрофон — 0% / -90.0 dB

Что сейчас слышит микрофон, в dB.

Шкала от -90 (тишина) до 0 (клиппинг/максимум)
-90.0 dB = тишина или микрофон не получает данных
При нормальной работе цифра должна «жить»: при громком звуке расти к -20…-10, при тихом — падать к -50…-40
Прогресс-бар показывает то же значение визуально

🔊 Колонки (monitor) — 0% / -90.0 dB

Что играет в самих колонках (берётся напрямую с sink.monitor).


Это «внутренний» звук системы — то, что отправляется на колонки до того, как воздух донесёт это до микрофона
Используется чтобы понять «играет ли вообще что-то» (для порога тишины)
-90.0 dB = сейчас ничего не воспроизводится в системе

🔉 Громкость sink — 43% / 43.0 %

Текущая установленная громкость выходного устройства.

То же значение, что вы видите в трее при клике на динамик
Программа меняет именно это значение

📊 Состояние — «Тишина»

Текстовый статус того, что делает программа сейчас:


«Тишина» — звук в колонках ниже порога, программа бездействует (как сейчас на скриншоте)
«err=+2.5 dB» — работает, разница между целью и текущим уровнем 2.5 dB (поднимает громкость)
«err=-1.2 dB» — работает, чуть громче цели (опускает громкость)
«🔒 Зафиксировано на 29%» — режим фиксации, всё в порядке
«🔒 Возврат к 29%» — режим фиксации, только что вернула после ручного вмешательства


Почему сейчас всё -90 dB и «Тишина»

На скриншоте:

Микрофон: -90 dB → не приходят данные с микрофона (parec не запущен или источник не тот)
Колонки (monitor): -90 dB → не приходят данные с monitor (тот же parec, или ничего не играет в системе)

Проверьте:

1.
Нажата ли кнопка «Старт»? Без неё захват аудио не идёт
2.
Если нажата — играет ли что-то в системе сейчас? Включите YouTube
3.
В терминале: pactl list short sources | grep -E "(input|monitor)" — есть ли там ваш микрофон и monitor


Как это всё работает вместе (краткая логика)

КАЖДЫЕ 0,5 сек:
  1. Берём 0,4 сек звука с микрофона → mic_dB
  2. Берём 0,4 сек с monitor          → play_dB
  3. Если play_dB < -55:               → «Тишина», стоп
  4. Иначе:
     err = -15.72 − mic_dB             ← насколько мы тише цели
     желаемая = текущая × 10^(err/20)
     новая = текущая + (желаемая − текущая) × 0.35   ← сглаживание
     ограничить шаг ±2%
     ограничить диапазоном 20%…56%
     применить
